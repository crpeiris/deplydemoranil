Group Project — ITC3380 (2025)

Project brief, milestones, and submission guidelines. Start early and iterate — use Git for collaboration.

Project brief

Form groups of 3–5. Build a small web application (Django, Flask, or similar) that solves a real user problem. Demonstrate planning, source control, and basic deployment steps.

Minimum requirements

  • User authentication (signup/login)
  • At least one CRUD model with list/detail views
  • Responsive UI and basic tests (unit or integration)
  • Source control history with meaningful commits and a short README

Milestones (suggested)

  1. Week 1 — Team formation, idea & rough sketch (1 page)
  2. Week 2 — Project planning: routes, models, tasks; initial repo & README
  3. Week 4 — Core functionality implemented (CRUD + auth)
  4. Week 6 — UI improvements, tests, documentation
  5. Week 8 — Final polish, deploy demo, prepare presentation

Submission guidelines

  • Push your final code to GitHub (or GitLab) and provide a repo link.
  • Ensure README contains setup instructions and a short project description.
  • Include a short video (2–5 minutes) or screenshots showing the app and key features.
  • Tag the instructor's account or share the link via the LMS before the deadline.

Git practice (recommended)

Use feature branches and PRs. Example commands:

git init
git checkout -b feature/your-feature
git add .
git commit -m "Add initial feature"
git push origin feature/your-feature
