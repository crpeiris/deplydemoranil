Group Project — ITC3380

This page describes the project brief, suggested milestones, and how to submit. Start early and iterate!

Project brief

Build a small, working software system that solves a real-world problem. Work in groups of 3–4. The project should demonstrate applied skills from the course: requirements analysis, design, implementation, testing, and deployment (or a demo).

Deliverables

  • Project report (PDF) — objectives, design, architecture, and evaluation.
  • Source code repository (GitHub/GitLab) with a clear README and setup steps.
  • A short demo video (5–8 minutes) or a live demo during the lab slot.

Assessment criteria

  • Functionality & correctness (40%)
  • Design & code quality (25%)
  • Testing & documentation (20%)
  • Presentation & demo (15%)

Milestones (suggested)

Split the work into short, testable milestones. Below is a suggested schedule — adapt to your team velocity and course timeline.

Week 1 — Proposal & Team setup

Choose idea, form teams, draft a 1-page proposal with objectives and main features.

Week 2 — Requirements & initial design

Write user stories/use cases, sketch the architecture, choose core technologies and data model.

Week 3–5 — Core implementation

Implement the main features, create automated tests, and push incremental commits to the repo.

Week 6 — Integration & testing

Polish features, fix bugs, add tests, and prepare a deployment/demo environment or recorded demo.

Week 7 — Documentation & final report

Write the project report, prepare slides, and finalise the README with setup/run instructions.

Submission guidelines

Follow these steps to submit your project correctly. Late penalties apply unless agreed with the instructor in advance.

  1. Repository: Push the final code to a public or private repo and grant the teaching team read access. Include a clear README with setup, run, and test instructions.
  2. Report: Upload the PDF report to the LMS assignment page. The report should include team member contributions and a short evaluation of limitations.
  3. Demo: Share the demo video link (YouTube unlisted or similar) or be prepared for the in-class live demo slot.
  4. Tagging: In the repo description add: ITC3380 - Group Project - 2025 - Team <X> and include a link to the report in the README.

Submission checklist

  • README with quickstart and dependency list
  • Tests and instructions to run them
  • Report PDF uploaded to LMS
  • Demo video link or demo time slot reserved
Submission Page (LMS)

Start early and iterate!

Begin small: deliver a minimal, working slice early and iterate. Frequent commits and short feedback loops reduce integration risk.

Tips for success
  • Keep a visible issue board (GitHub Issues or similar) and track tasks.
  • Split features into small PRs and review each other's code.
  • Write at least a few automated tests for critical paths.
  • Document how to run the project in 3 steps or less in the README.
  • Communicate weekly and set a demo milestone at least one week before submission.

Questions? Contact the teaching team via the LMS or email the course coordinator.
